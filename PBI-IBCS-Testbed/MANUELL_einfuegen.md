# MANUELL einfuegen — Fallback (falls das PBIP nicht sauber oeffnet)

Diese Datei ist der manuelle Plan B. Alle Daten sind identisch zum PBIP.

> **Wichtiger Hinweis zur Datumstabelle:** `DimDate` ist bewusst **taeglich & lueckenlos** (2024-01-01 … 2025-12-31), nicht 24 Monatszeilen. Power BI laesst *Als Datumstabelle markieren* und `SAMEPERIODLASTYEAR`/`DATEADD` nur auf einer tagesgenauen, zusammenhaengenden Datumsspalte zu. Die Fakten liegen auf dem Monatsersten; die Spalte `YearMonth` liefert die 24 Achsenpunkte fuer Zeitreihen-Visuals.

## A · Power-Query (M) — eine Query je Tabelle

Erweiterter Editor (Daten transformieren → Neue Quelle → Leere Abfrage → Erweiterter Editor) und je Block einfuegen. Query exakt wie den Tabellennamen benennen.

### DimDate

```m
let
    Start = #date(2024,1,1),
    DayCount = Duration.Days(#date(2025,12,31) - #date(2024,1,1)) + 1,
    Dates = List.Dates(Start, DayCount, #duration(1,0,0,0)),
    T0 = Table.FromList(Dates, Splitter.SplitByNothing(), {"Date"}),
    T1 = Table.TransformColumnTypes(T0, {{"Date", type date}}),
    Monate = {"Jan", "Feb", "Mär", "Apr", "Mai", "Jun", "Jul", "Aug", "Sep", "Okt", "Nov", "Dez"},
    T2 = Table.AddColumn(T1, "Year", each Date.Year([Date]), Int64.Type),
    T3 = Table.AddColumn(T2, "MonthNo", each Date.Month([Date]), Int64.Type),
    T4 = Table.AddColumn(T3, "MonthName", each Monate{Date.Month([Date]) - 1}, type text),
    T5 = Table.AddColumn(T4, "Quarter", each "Q" & Text.From(Date.QuarterOfYear([Date])), type text),
    T6 = Table.AddColumn(T5, "YearMonth", each Text.From(Date.Year([Date])) & "-" & Text.PadStart(Text.From(Date.Month([Date])), 2, "0"), type text),
    T7 = Table.AddColumn(T6, "IstAktuell", each Date.Year([Date]) = 2025, type logical)
in
    T7
```

## B · Beziehungen (Modellansicht anlegen, alle 1:* von der Dimension)

- `DimDate[Date]` 1 — * `FactActuals[Date]`, `FactPlan[Date]`, `FactBudget[Date]`, `FactForecast[Date]`
- `DimProduct[ProductKey]` 1 — * gleiche vier Faktentabellen
- `DimRegion[RegionKey]` 1 — * gleiche vier Faktentabellen
- `FactDaily` bleibt **ohne** Beziehung (Tagesgranularitaet, eigene KPI-Logik)
- Danach: `DimDate` markieren via *Tabellentools → Als Datumstabelle markieren → Date*

## C · DAX-Measures (Tabelle `_Measures`)

```dax
AC = SUM ( FactActuals[Amount] )
PY = CALCULATE ( [AC], SAMEPERIODLASTYEAR ( DimDate[Date] ) )
PL = SUM ( FactPlan[Amount] )
BU = SUM ( FactBudget[Amount] )
FC = SUM ( FactForecast[Amount] )
FC_FY = [AC] + [FC]
ΔPY = [AC] - [PY]
ΔPY% = DIVIDE ( [AC] - [PY], [PY] )
ΔPL = [AC] - [PL]
ΔPL% = DIVIDE ( [AC] - [PL], [PL] )
ΔBU = [AC] - [BU]
ΔBU% = DIVIDE ( [AC] - [BU], [BU] )
Umsatz = SUM ( FactActuals[Amount] )
Kosten = SUM ( FactActuals[Cost] )
Marge = [Umsatz] - [Kosten]
Marge% = DIVIDE ( [Umsatz] - [Kosten], [Umsatz] )
Orders_AC = CALCULATE ( SUM ( FactDaily[Value] ),   FactDaily[KPI] = "Orders" )
Orders_PY = CALCULATE ( SUM ( FactDaily[ValuePY] ), FactDaily[KPI] = "Orders" )
Revenue_AC = CALCULATE ( SUM ( FactDaily[Value] ),   FactDaily[KPI] = "Revenue" )
Revenue_PY = CALCULATE ( SUM ( FactDaily[ValuePY] ), FactDaily[KPI] = "Revenue" )
```

Formatierung: Betraege `#,##0` (de-DE → 1.234.567), Prozente `0.0%`.
